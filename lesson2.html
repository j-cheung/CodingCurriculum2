Lesson 3 Teacher's Notes
1. Introduce the Concept of “If Statements”
2. Introduce the Concept of “Functions” and why they are necessary
3. Building an Engduino program of running lights using if statements and functions
Activity:
Students will build an Engduino program via TouchDevelop – through this exercise they will be introduced to the concepts of “if statements” and “functions” along the way. The students are reminded of the LEDs on an Engduino work and are used before going on the next step of creating a function.
Functions are then introduced to the students on how to split code into more readable chunks. Within the function – if statements are then introduced to show how some code can be run only if specific conditions are met.
If statements
If statements are conditional statements that will only be run if specific conditions are true. For example you could have an if statement
“If (10 < 100): do something”
You could also have an in statement that has multiple conditions such as:
“If (10 < 100 AND 100 < 1000): do something”
Since these conditions are true, therefore the code is executed.
If statements also have a segment called “else” – this indicates that if the conditions for the if statement are not met, then the code after the else segment is executed.
“If (10 > 100): do something
Else: do something else”
Functions
Functions split code into more readable chunks and allows the user to re-use code when required.
For example: add is a function that takes in two variables and returns a variable that is the sum of both variables.
“def add(integer number1, integer number2):
Return (number1 + number2)”
In this particular case, you have to call add with 2 variables such as add(7, 8) which will then add up both the variables and return their sum, 15
